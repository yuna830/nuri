## 사용 목적

경찰청 실종자 검색 데이터를 조회하여 서비스 내 실종자 정보 목록과 상세 정보 제공에 활용한다.

실종자 검색 API는 안전Dream 사이트에서 제공하는 실종자 검색 서비스를 Open API로 조회하는 기능이며, 사용자, 보호자, 복지사가 실종자 정보를 확인할 수 있도록 한다.

## 데이터 출처

경찰청 안전Dream Safe182

## 제공 기관

경찰청

## 서비스명

경찰청_실종자검색_API

## 데이터 형식

XML

## Endpoint

```txt
https://www.safe182.go.kr/api/lcm/findChildList.do
```

## Endpoint + 세부 경로

```txt
https://www.safe182.go.kr/api/lcm/findChildList.do
```

## 관련 템플릿 URL

```txt
https://www.safe182.go.kr/api/lcm/findChildListT.do
```

템플릿 URL은 AJAX로 호출한 결과를 HTML 형태로 삽입하는 방식이다.  
서버 DB 저장이나 RAG 수집에는 Open API URL인 `findChildList.do`를 사용한다.

## 사용하는 세부 기능

|기능|경로|설명|
|---|---|---|
|실종자 목록 검색|/findChildList.do|조건에 따라 실종자 검색 목록 조회|
|실종자 검색 템플릿 조회|/findChildListT.do|HTML 템플릿 형태로 실종자 검색 결과 표시|

## 요청 방식

POST

## 인증 방식

경찰청 Safe182 API 인증 정보를 사용한다.

```txt
esntlId={SAFE182_ESNTL_ID}
authKey={SAFE182_AUTH_KEY}
```

주의: 인증키 원문은 Obsidian에 작성하지 않는다.

## 데이터 처리 흐름

1. 서버에서 경찰청 Safe182 실종자 검색 API를 호출한다.
    
2. XML 응답을 수신한다.
    
3. 한글 인코딩 문제를 보정한다.
    
4. 실종자 이름, 성별, 발생일자, 발생장소, 인상착의 등 필요한 필드를 추출한다.
    
5. 사진 Base64 데이터가 있으면 이미지 파일로 저장한다.
    
6. DB에는 사진 원문이 아니라 저장 경로를 저장한다.
    
7. externalKey 기준으로 중복 여부를 확인한다.
    
8. 기존 데이터가 있으면 갱신하고, 없으면 새로 저장한다.
    
9. syncedAt 값을 현재 시간으로 갱신한다.
    

## DB 저장 기준

경찰청 실종자 데이터는 `police_missing_alerts` 테이블에 저장한다.

이름, 발생일자, 발생주소, 성별을 조합한 externalKey를 기준으로 중복 저장을 방지한다.

```txt
externalKey = 이름 + 발생일자 + 발생주소 + 성별
```

원본 응답 전체를 저장하기보다 서비스 제공에 필요한 필드 중심으로 저장한다.

사진 데이터는 Base64 원문을 DB에 저장하지 않고, 이미지 파일로 저장한 뒤 파일 경로만 DB에 저장한다.

## 주의사항

인증키 원문은 문서에 저장하지 않는다.

실종자 데이터는 개인정보성 정보가 포함될 수 있으므로 Obsidian과 RAG에 원본 데이터를 대량 저장하지 않는다.

사진 Base64 데이터는 Obsidian에 저장하지 않는다.

API 응답 필드는 비어 있을 수 있으므로 null 처리가 필요하다.

경찰청 API 응답은 한글 인코딩 문제가 발생할 수 있으므로 서버에서 인코딩 보정 로직이 필요하다.

실종자 검색 API는 조건 검색과 페이지네이션이 가능하므로 조회 조건과 페이지 번호를 함께 관리해야 한다.

일일 조회 건수 제한이 있으므로 무분별한 반복 호출을 피해야 한다.

## 최종 정리

이 API는 경찰청 실종자 검색 데이터를 조회하여 서비스 내 실종자 정보 제공에 활용하는 데이터 소스다.

DB에는 실종자 식별에 필요한 주요 필드와 사진 저장 경로를 저장하고, RAG에는 실종자 원본 데이터가 아니라 API 구조, 필드 의미, 처리 흐름, 관리 규칙만 포함한다.
