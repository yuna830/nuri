## 목적

외부 API 인증키, 기관 ID, authKey, serviceKey 등 민감한 인증 정보를 안전하게 관리하기 위한 기준을 정리한다.

이 문서는 Obsidian, GitHub, 코드 저장소, 서버 설정 파일에서 인증 정보를 어떻게 다룰지 정의한다.

## 기본 원칙

1. 인증키 원문은 Obsidian에 저장하지 않는다.

2. 인증키 원문은 GitHub에 올리지 않는다.

3. 인증키는 서버 환경변수 또는 application.properties에서 관리한다.

4. 문서에는 실제 값 대신 변수명만 작성한다.

5. 테스트 요청 URL에도 실제 인증키를 남기지 않는다.
