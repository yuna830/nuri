지역별 지자체 복지서비스 목록과 상세 정보를 조회하여 사용자 상황에 맞는 복지제도 추천에 활용한다.

## 데이터 형식

## 인증 방식

공공데이터포털 일반 인증키를 사용한다.

```txt
serviceKey=${DATA_GO_KR_SERVICE_KEY}
```

주의:

- 인증키 원문은 Obsidian에 작성하지 않는다.
    
- GitHub에 올라가는 파일에도 실제 인증키를 작성하지 않는다.
    
- 인증키는 서버 환경변수 또는 application.properties에서 관리한다.

## 데이터 처리 흐름

1. 목록 조회 API를 호출하여 복지서비스 목록을 가져온다.
	
2. 목록 응답에서 servId를 추출한다.
	
3. servId를 이용해 상세 조회 API를 호출한다.
	
4. XML 응답을 수신한다.
	
5. 복지서비스명, 대상, 지원내용, 신청방법 등 필요한 필드를 추출한다.
	
6. DB 또는 RAG 문서 저장소에 저장한다.
	
7. RAG 검색용 Markdown 문서로 변환한다.
	
8. raw_hash 비교로 변경 여부를 확인한다.
	
9. 변경된 문서만 Qdrant에 재임베딩한다.

## DB 저장 기준

servId를 기준으로 중복 저장을 방지한다.

servId가 없는 경우에는 서비스명, 소관 기관명, 담당 기관명, 상세 링크를 조합하여 중복 여부를 판단한다.

현재 복지서비스 전용 테이블이 없다면, 우선 RAG 검색용 문서로 변환하여 rag_documents 테이블에 저장하고, 수집 작업 이력은 rag_ingest_jobs 테이블에서 관리할 수 있다.

추후 복지서비스를 화면에서 목록, 상세, 필터링 형태로 직접 제공해야 한다면 별도 테이블을 추가할 수 있다.

예상 추가 테이블:

welfare_services
welfare_service_details

## 주의사항

인증키 원문은 문서에 저장하지 않는다.

API 응답 필드는 비어 있을 수 있으므로 null 처리가 필요하다.

복지서비스 상세 정보는 목록 조회만으로 부족할 수 있으므로 상세 조회 API와 함께 사용한다.

RAG 문서에는 사용자가 이해할 수 있는 설명 중심으로 변환한다.

## 최종 정리

이 API는 사용자에게 지역별 복지서비스를 추천하거나 설명하기 위한 핵심 데이터 소스다.

DB에는 서비스 식별자와 주요 필드를 저장하고, RAG에는 사용자가 질문할 만한 복지서비스 설명 중심으로 변환한다.
